# Sentynyx macOS release checklist

Everything needed to go from "green CI" to "signed, notarized `.dmg` users
can download and open without Gatekeeper warnings".

> **Open-source build.** Public releases are built with the **default** Cargo
> features — no `team-cloud`, no telemetry. (The commercial edition is built
> separately with `--features team-cloud`.) Auto-update is off for the OSS
> release: `bundle.createUpdaterArtifacts` is `false`, so the `TAURI_SIGNING_*`
> secrets below are **not required** — set them up only when you turn the
> updater on.

## One-time setup (on the machine that will hold the release credentials)

1. **Apple Developer Program**: active membership at developer.apple.com ($99/yr).
2. **Developer ID Application cert**:
   - Xcode → Settings → Accounts → your team → Manage Certificates → `+` → "Developer ID Application".
   - Or: developer.apple.com → Certificates → `+` → "Developer ID Application" → follow CSR flow.
3. **App-specific password** for notarization:
   - appleid.apple.com → Sign-In & Security → App-Specific Passwords → Generate.
   - Save it in your password manager. Label: "sentynyx notarytool".
4. **Team ID**: developer.apple.com/account → Membership → Team ID (10 chars).

## Add the following GitHub repo secrets

Repo → Settings → Secrets and variables → Actions → New repository secret.

| Secret | Source |
|---|---|
| `APPLE_CERTIFICATE` | `base64 -i DeveloperIDApplication.p12 \| pbcopy` (export from Keychain Access) |
| `APPLE_CERTIFICATE_PASSWORD` | The password you set when exporting the .p12 |
| `APPLE_SIGNING_IDENTITY` | e.g. `Developer ID Application: Your Name (TEAMID12345)` — find with `security find-identity -v -p codesigning` |
| `APPLE_ID` | Your Apple ID email |
| `APPLE_PASSWORD` | The app-specific password from step 3 |
| `APPLE_TEAM_ID` | The 10-char Team ID |

Optional (for the `release` workflow's auto-update signing, Phase 2.5):

| Secret | Source |
|---|---|
| `TAURI_SIGNING_PRIVATE_KEY` | Ed25519 private key generated by `tauri signer generate` |
| `TAURI_SIGNING_PRIVATE_KEY_PASSWORD` | Its passphrase |

## Cut a release

```bash
# Bump version in apps/desktop/src-tauri/tauri.conf.json AND
# apps/desktop/src-tauri/Cargo.toml AND apps/desktop/package.json.
# They should all agree.
git tag v0.4.0
git push origin v0.4.0
```

GitHub Actions runs the `release` workflow:

1. Builds the universal NER sidecar.
2. Imports the Apple cert into an ephemeral keychain.
3. Runs `pnpm tauri build --target universal-apple-darwin` with the bundling
   env vars set — this compiles both Rust binaries universal, bundles the
   vendored ONNX dylib into `Sentynyx.app/Contents/Frameworks/`, copies the
   NER sidecar into `Sentynyx.app/Contents/MacOS/sentynyx-ner`, and runs
   `codesign` with the Developer ID + entitlements.
4. Runs `xcrun notarytool submit --wait` on the resulting DMG. Waits for
   Apple's notary service (usually 1–10 min).
5. Runs `xcrun stapler staple` so the ticket travels with the DMG.
6. Uploads `.dmg` as a workflow artifact. Download from the run page.

## Smoke-test the output on a clean Mac

```bash
# On a Mac that hasn't built Sentynyx locally (or use a fresh account):
spctl --assess --type execute --verbose Sentynyx.app
# Expected: "Sentynyx.app: accepted"

# If you see "unidentified developer" when opening the DMG:
# - Check that notarization succeeded (xcrun notarytool log <submission-id>)
# - Check that the DMG has a stapled ticket (`stapler validate`)

# Launch and verify:
open Sentynyx.app
# - Window opens without a codesign warning
# - Settings → API keys → Save → "saved to OS keychain" (not "protected file")
# - Enter a prompt with a name → NER tag appears in the Vendetta panel
#   (proves the sidecar + ORT dylib are wired correctly inside the .app)
```

## Troubleshooting

**`codesign --deep --verify` complains about an unsigned dylib.**
The vendored ORT dylib must ship signed. Tauri signs everything under
`Frameworks/` automatically when `APPLE_SIGNING_IDENTITY` is in env.
If you re-vendor an ORT dylib, re-run the release workflow end-to-end.

**Notarization rejects with "the executable does not have the hardened runtime enabled".**
`bundle.macOS.hardenedRuntime` is set in `tauri.conf.json`. If you see
this error, confirm the Tauri build actually picked up that config
(some Tauri versions silently drop unknown fields — check the bundle
output log for `hardenedRuntime` mentions).

**Notarization rejects with "the binary uses allocate-exec without the
allow-unsigned-executable-memory entitlement".**
Entitlements are in `entitlements.plist`. If you touch llama.cpp's build
options, make sure you keep `com.apple.security.cs.allow-jit` and
`com.apple.security.cs.allow-unsigned-executable-memory`.

## What still requires manual testing

- Pre-merge: run a tag-push in a scratch branch with your secrets set; the
  notary round-trip doesn't have a dry-run mode, so you'll burn a submission
  slot.
- Pre-release: download the artifact onto a Mac that has NEVER run Sentynyx
  before, drag `.app` from the DMG to /Applications, double-click, confirm
  no warnings.
